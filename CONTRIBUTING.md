# Contributing to the Untitled Protocol

Thank you for your interest in the Untitled Protocol.

## Ways to contribute

### Report a spec issue

If you find an ambiguity, conflict, or gap in the specification, open a GitHub issue describing:
- The section and text in question
- What the ambiguity is
- What behavior you expected

### Submit a tool or integration

Built something that reads, writes, or respects the Untitled Protocol? Open an issue using the [tool submission template](https://github.com/untitled-now/protocol/issues/new?template=tool-submission.md&title=Tool+submission). Include:

- Tool name and a one-line description
- Which states it supports
- A link (GitHub repo, npm package, or hosted URL)

Accepted tools are listed at [untitled.now/tools/](https://untitled.now/tools/).

### Propose a spec change

The protocol is v0.1 and deliberately minimal. If you want to propose an addition or change:

1. Open an issue first — describe the problem you're solving, not the solution.
2. If there's agreement the problem is in scope, submit a pull request editing `SPEC.md`.
3. Changes to the controlled vocabulary (the six tokens) require strong justification. The vocabulary is intentionally small.

### Pull requests

- Keep PRs focused. One change per PR.
- For `SPEC.md` edits, also update the Changelog section at the bottom.
- For `README.md` edits, no changelog entry needed.

## Code of conduct

Be direct. Be honest. Assume good faith. Don't be a jerk.
